02. Contexte & Problématique

L'Entreprise

MédiaCom Horizon est une agence de communication digitale en pleine expansion située à Bordeaux. Elle compte 45 collaborateurs répartis en 3 pôles distincts : Administratif, Créatif et un espace de Coworking pour les clients.

L'infrastructure actuelle repose sur un réseau "à plat" (Flat Network) hérité de la création de l'entreprise.

La Problématique

"Comment garantir la sécurité des données financières tout en offrant une connectivité fluide aux créatifs et aux invités, dans un réseau actuellement saturé et non segmenté ?"

  • Broadcast Storms fréquents.
  • Sécurité nulle (Invités accèdent à la Compta).
  • Pénurie d'adresses IP.

05. Configuration N2 : VLANs

La première étape consiste à déclarer les VLANs dans la base de données du commutateur et à assigner les ports d'accès.

  • VLAN 10 : Ports 2 à 10 (Admin)
  • VLAN 20 : Ports 11 à 20 (Créatif)
  • VLAN 99 : Ports 21 à 24 (Invités/Wi-Fi)
SW-MEDIACOM - SSH
Switch(config)# vlan 10
Switch(config-vlan)# name ADMIN
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name CREATIF
Switch(config-vlan)# exit
! Configuration des ports d'accès
Switch(config)# interface range fa0/2 - 10
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10

06. Configuration N2 : Trunk 802.1Q

Le port Gi0/1 relié au routeur doit transporter les trames de tous les VLANs. Le protocole 802.1Q ajoute un "Tag" (étiquette) à la trame Ethernet pour identifier son VLAN d'origine.

SW-MEDIACOM - SSH
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# description UPLINK_ROUTER
! Force le mode Trunk (pas de négo DTP)
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport nonegotiate
! Sécurité : Autoriser uniquement les VLANs actifs
Switch(config-if)# switchport trunk allowed vlan 10,20,30,99

07. Configuration N3 : Router-on-a-Stick

R-MEDIACOM - SSH
Router(config)# interface gi0/0/1
Router(config-if)# no shutdown
Router(config-if)# exit

! Sous-interface ADMIN (VLAN 10)
Router(config)# interface gi0/0/1.10
Router(config-subif)# encapsulation dot1q 10
Router(config-subif)# ip address 192.168.10.254 255.255.255.0

! Sous-interface CREATIF (VLAN 20)
Router(config)# interface gi0/0/1.20
Router(config-subif)# encapsulation dot1q 20
Router(config-subif)# ip address 192.168.20.254 255.255.255.0

Principe

Le routeur utilise une seule interface physique (Gi0/0/1) découpée en plusieurs **sous-interfaces logiques**.

Chaque sous-interface :

  • Est associée à un ID de VLAN (commande encapsulation dot1q).
  • Possède une adresse IP qui servira de Passerelle par défaut pour les clients de ce VLAN.

08. Services Réseau : DHCP

Le routeur agit comme serveur DHCP centralisé. Il distribue automatiquement les adresses IP aux clients en fonction du VLAN d'origine de la requête.

! Pool pour les Créatifs
Router(config)# ip dhcp pool POOL_CREATIF
Router(dhcp-config)# network 192.168.20.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.20.254
Router(dhcp-config)# dns-server 8.8.8.8

! Pool pour les Invités
Router(config)# ip dhcp pool POOL_GUEST
Router(dhcp-config)# network 192.168.99.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.99.254

09. Sécurité : Access Control Lists

Objectif Critique

Interdire strictement l'accès au VLAN 10 (Admin) depuis le VLAN 99 (Invité), tout en laissant les invités accéder à Internet.

Nous utilisons une ACL Étendue numérotée 100. Elle est appliquée en entrée (in) sur l'interface des invités pour filtrer le trafic dès son arrivée sur le routeur.

! 1. Deny Guest to Admin
Router(config)# access-list 100 deny ip 192.168.99.0 0.0.0.255 192.168.10.0 0.0.0.255

! 2. Permit Internet (Any)
Router(config)# access-list 100 permit ip any any

! 3. Apply to Interface
Router(config)# interface gi0/0/1.99
Router(config-subif)# ip access-group 100 in
